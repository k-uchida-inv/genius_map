# 課題仮説シート

> A4 1枚に収まるように記入してください。

## プロダクト名

GeniusMap
https://bizfriend.vercel.app/

## 一言で言うと？

思考整理・企画立案をするナレッジワーカーが、AIの連想・分析・調査によってマインドマップの発想を無限に広げられるサービス

## 1. 誰の課題か？（ターゲットユーザー）

- **どんな人？**：個人で情報整理・企画立案・学習を行うナレッジワーカー。リサーチャー、ライター、プランナー、学生。従来のマインドマップツール（MindMeister, Miro）を使ったことがあるが、発想の広がりに限界を感じている人。
- **人数の規模感**：マインドマップツールのグローバル市場は数百万ユーザー規模。日本国内でも MindMeister / XMind / Miro 利用者は推定数十万人。身の回りに5〜10人はいるレベル。
- **具体的に1人思い浮かべてください**：30代・IT企業のプランナー。新規事業の企画で週3回はマインドマップを使うが、いつも同じキーワードの周辺をぐるぐる回ってしまう。「もっと広い視点がほしい」と感じている。

## 2. どんな課題か？

- **困っていること**：マインドマップを作る際、自分の知識の範囲内でしか連想できず、思考が閉じてしまう。調べ物をするたびにブラウザとマインドマップツールを行ったり来たりする。作ったマップを俯瞰して「結局何が言えるのか」を整理するのに時間がかかる。
- **今どうやって対処しているか**：ChatGPTに「◯◯の関連トピックを列挙して」と聞き、手動でマインドマップに貼り付けている。Google検索結果を見ながらノードを追加している。つまり、複数ツールを跨いだ手作業で我慢している。
- **なぜそれでは不十分か**：ツール間のコンテキストスイッチで思考が途切れる。AIの出力をマップ構造に整形する手間が大きい。連想→調査→分析→要約の一連のフローがシームレスに繋がっていない。

## 3. なぜ今この課題を解くのか？

- 最近の環境変化（技術・法律・社会など）：LLMの性能向上により、キーワード連想・関係性分析・トピック調査がAPI一本で実現可能になった。Claude / GPT-4レベルのモデルが安価に使えるようになり、リアルタイムなAI連携が現実的なコストで実装できる。
- 自分自身の原体験：企画・設計の思考整理でマインドマップを日常的に使うが、「ここから先が出てこない」という壁を毎回感じる。ChatGPTで連想させた結果を手でコピペする作業に非効率さを感じていた。
- 既存の解決策が失敗している理由：MindMeister等の既存ツールはAI連携が後付けで、マップの文脈を理解した連想ができない。Miroは汎用的すぎてマインドマップ特化の体験が弱い。AIネイティブなマインドマップツールはまだブルーオーシャン。

## 4. どう解決するか？

- **解決アプローチ（1-2文）**：マインドマップ上で直接AIが連想・分析・調査を行い、ノードとして結果をマップに追加する。思考→拡張→整理→要約の全フローがマップキャンバス上で完結する。
- **モックで見せるコア機能**：
  - ノードホバーで「AI Associate」ボタン → 関連キーワードが子ノードとしてポンポン追加される
  - ノード選択でサイドパネル「AI Research」 → トピックの調査結果がストリーミング表示
  - マーキングした複数ノードを「Summarize」 → AIがアイデアとして統合・要約
  - 「AI Analyze」 → マップ全体の関係性を分析
- **ユーザーがこのプロダクトを使ったら何が変わるか**：「思いつかない」がなくなる。1つのキーワードからAIが枝を伸ばし、知らなかった視点を提示する。調べ物のためにブラウザに移動する必要がなくなる。マップを作った後の「で、結局何が言えるの？」をAIが即座に言語化してくれる。
